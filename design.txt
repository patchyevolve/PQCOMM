0. Goals

The system must provide:

Real-time duplex communication
Low latency audio transmission
Secure chat and file transfer
Exclusive session tunnel
Strong identity verification
Post-quantum safe handshake
Low CPU usage in trusted mode
Resistance to spoof, flood, replay
Ability to survive packet loss / delay
Ability to adapt under jamming
Kernel-level filtering support
Hardware crypto acceleration support
Secure key storage
Automatic key rotation
Multi-path transport
Relay / mesh support
CLI control interface

Non-goals:

anonymity network
internet routing replacement
RF hardware layer protection
defeating physical jamming completely
1. Design Philosophy

The system follows the trusted pipe model.

Rules:

Trusted packets must move through minimal path.
Unknown packets must never reach session.
Outer layers protect link.
Inner layers carry data.
Transport may change, session must not.

Principles:

Gate before decrypt
Decrypt before channel
Scheduler before encrypt
Control before audio
Audio before file
Outer layers must not see plaintext
Session must be exclusive
Handshake must run once
Fast path must be constant time

Not guaranteed protection against:

ISP blocking
local OS compromise
hardware key theft
